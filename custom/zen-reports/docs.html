Zen Reports — Technical Documentation

Every section's data source, GA4 API calls, and calculation math. No AI processing — everything is GA4 Data API + basic arithmetic.

1GA4 Data API — Base Setup

Endpoint: POST /v1beta/properties/{property_id}:runReport

Auth: OAuth 2.0 with analytics.readonly scope.

LLM Source Filter — used across all sections:

{
  "dimensionFilter": {
    "orGroup": {
      "expressions": [
        { "filter": { "fieldName": "sessionSource",
          "stringFilter": { "matchType": "CONTAINS", "value": "chatgpt" } } },
        { "filter": { "fieldName": "sessionSource",
          "stringFilter": { "matchType": "CONTAINS", "value": "gemini.google" } } },
        { "filter": { "fieldName": "sessionSource",
          "stringFilter": { "matchType": "CONTAINS", "value": "claude.ai" } } },
        { "filter": { "fieldName": "sessionSource",
          "stringFilter": { "matchType": "CONTAINS", "value": "perplexity" } } },
        { "filter": { "fieldName": "sessionSource",
          "stringFilter": { "matchType": "CONTAINS", "value": "copilot.microsoft" } } }
      ]
    }
  }
}
Source strings may vary (e.g., chat.openai.com vs chatgpt.com). Maintain a lookup table mapping known source strings to platform names.

2Time Range & Comparison

Monthly-only ranges: Last 1, 3, 6, or 12 months.

current_end = today
current_start = today - (selected_months × 30)
previous_end = current_start - 1
previous_start = previous_end - (selected_months × 30)

GA4 supports dual date ranges natively:

{
  "dateRanges": [
    { "startDate": "2024-09-16", "endDate": "2025-03-14", "name": "current" },
    { "startDate": "2024-03-16", "endDate": "2024-09-15", "name": "previous" }
  ]
}

For "Same period last year":

previous_start = current_start - 365 days
previous_end = current_end - 365 days

Percentage change:

change_pct = ((current - previous) / previous) × 100
If previous = 0, display "New" instead of percentage

3Total Sessions + Per-LLM Cards GA4 API

{
  "dimensions": [{ "name": "sessionSource" }],
  "metrics": [{ "name": "sessions" }],
  "dateRanges": [ /* current + previous */ ],
  "dimensionFilter": { /* LLM source filter */ }
}

Processing: Group rows by mapped platform name → sum sessions per platform for each period → compute change %.

Sparkline data — separate call with monthly granularity:

{
  "dimensions": [
    { "name": "yearMonth" },
    { "name": "sessionSource" }
  ],
  "metrics": [{ "name": "sessions" }]
}

5LLM Share Donut GA4 API

No extra API call — uses per-platform totals from Section 3.

6AI Visibility Score Calculated

No extra API call. Calculated from existing data:

volume_score     = min(total_sessions / 15, 40)         // max 40 pts
diversity_score  = active_platform_count × 4           // max 20 pts
growth_score     = max(0, min(change_pct × 0.5, 20))  // max 20 pts
engagement_score = min(avg_session_minutes × 2, 20)    // max 20 pts

ai_visibility_score = min(volume + diversity + growth + engagement, 100)
Scaling note: The volume divisor (15) is for small-to-mid sites. For high-traffic sites, use logarithmic: volume = min(log10(sessions + 1) × 12, 40)

7Insight Cards Calculated

Pure conditional logic — no AI, no extra API calls.

Insight 1: Dominant platform

dominant = platform with max sessions
pct = (dominant.sessions / total) × 100
change = ((current - previous) / previous) × 100

→ "{name} drives {pct}% of all AI traffic with {sessions} sessions — up/down {change}% from {compare_period}."

Insight 2: Best engagement

best = platform with highest averageSessionDuration

→ "{name} users have the highest avg session time across your pages."

Insight 3: Biggest decline (or fastest grower)

declines = platforms where current < previous
worst = decline with largest absolute change %

→ "{name} referral traffic is down {change}% this period ({prev} → {current} sessions)."

If no declines → show fastest grower (excluding dominant)
If only 1 platform has traffic → "Optimize for other LLMs to diversify."

8LLM Engagement Quality GA4 API

{
  "dimensions": [{ "name": "sessionSource" }],
  "metrics": [
    { "name": "averageSessionDuration" },
    { "name": "screenPageViewsPerSession" },
    { "name": "bounceRate" }
  ],
  "dimensionFilter": { /* LLM source filter */ }
}

Format: averageSessionDuration in seconds → Xm Ys. bounceRate returned as decimal (0.32 → 32%).

Color coding: Green ≤20%, Amber 21–35%, Red >35%.

9Top Performing Pages GA4 API

{
  "dimensions": [
    { "name": "sessionSource" },
    { "name": "pagePath" }
  ],
  "metrics": [
    { "name": "sessions" },
    { "name": "averageSessionDuration" },
    { "name": "bounceRate" }
  ],
  "orderBys": [{ "metric": { "metricName": "sessions" }, "desc": true }],
  "limit": "100"
}

When page filter active, add pagePath EXACT match to dimensionFilter's andGroup.

10Geographic Breakdown GA4 API

// Countries
{ "dimensions": [{ "name": "country" }], "metrics": [{ "name": "sessions" }] }

// Cities
{ "dimensions": [{ "name": "city" }], "metrics": [{ "name": "sessions" }], "limit": "10" }

Map rendering: D3.js + world-atlas@2/countries-110m.json. Handle name aliases (e.g., "United States of America" → "United States").

11Browsers GA4 API

{ "dimensions": [{ "name": "browser" }], "metrics": [{ "name": "totalUsers" }] }
browser_pct = (browser_users / total_users) × 100
Group <1% into "Others"

13Page-Level Filter

When active, every API call adds this to the andGroup:

{ "filter": { "fieldName": "pagePath",
  "stringFilter": { "matchType": "EXACT", "value": "/selected-page" } } }

Page list populated on init (1 call, limit 1000, sorted by sessions). Search is client-side string matching.
